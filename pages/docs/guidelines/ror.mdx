# Ruby on Rails

## Standard Gems

Here, we plan to get insights into different gems used for Ruby on Rails application development phase, testing phase and user interface (UI) changes along with the installation process to add those gems into our projects.

### Development

While developing a rails application at Truemark, we place great emphasis on user authentication and authorization so we prefer using the following modules to ensure the preceding conditions are met.

<br />

#### Devise

Devise is a flexible authentication module for Rails with Warden. It is a rack based MVC solution composed of multiple modules which can be signed into or added to our projects based on the use cases.

Devise can be added to any project by either executing the global install command or adding the devise gem in the gemfile.

To `global install` devise

```
gem install devise
```

To add the module in the `gemfile`

```
gem 'devise', '~> 4.8', '>= 4.8.1'
```

The aforementioned devise version is subject to change so please find the official devise gem and github link below for latest version:

Official Documentation: [devise docs](https://github.com/heartcombo/devise)

Rubygems: [devise gem](https://rubygems.org/gems/devise)

<br />

### Testing

After a feature has been completed, it has to be documented and tested to ensure that the code works as expected for a behavior and use case. It always starts with writing a failing test case and then refactoring just enough code to pass the test.

### User Interface (UI)

Ruby on Rails is a compelling framework to develop web applications due to its support for MVC architectural pattern that separates the entire application into three main logical components: Model, View and Controller. It's the View layer that handles the user interface of the application which is a functional structure when it comes to enhancing the user interaction.

<br />

This gem can be added to any rails project by either executing the global install command or adding the pagy gem in the gemfile.
